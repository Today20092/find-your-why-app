# Domain docs

This is a single-context repository.

Before exploring the codebase, read root `CONTEXT.md` when present and relevant ADRs under `docs/adr/`. If either is absent, proceed silently; domain-modeling skills create them only when needed.

Use the glossary vocabulary from `CONTEXT.md` in issues, tests, and implementation. If work contradicts an ADR, identify the conflict explicitly rather than silently overriding the decision.
